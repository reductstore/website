Hello, everyone!

Starting with **ReductStore v1.19**, we are changing how we license and package the project. From this version onward, ReductStore is split into two editions: **[ReductStore Core](https://github.com/reductstore/reductstore)** and **ReductStore Pro**.

{/* truncate */}

Previously, ReductStore shipped as a single edition under **BUSL-1.1 (Business Source License)**. With v1.19, the core database is open source under Apache 2.0, while Pro continues under commercial terms.

## What's changing

With this release, **[ReductStore Core](https://github.com/reductstore/reductstore)** becomes the open-source foundation of the project and is now available under the **Apache License 2.0**. It includes the database server and the core functionality that most users rely on for edge deployments and everyday data workflows.

Alongside Core, we will continue offering **ReductStore Pro**, which is distributed under a **commercial license**. The Pro edition provides additional capabilities and support options for teams with more advanced requirements.
See **[Pricing](/pricing)** for a clear comparison of what is included in each edition.

## Why we're doing this

We want to better support open source communities and make it easier to contribute to, integrate with, and build on top of **[ReductStore Core](https://github.com/reductstore/reductstore)**. Just as importantly, we want users to be able to run ReductStore on the edge without licensing restrictions for the core database use cases.

At the same time, we want to keep a sustainable commercial model for companies that build more complex cloud setups or need advanced support and functionality around robotics and IIoT data formats. That is the role of ReductStore Pro.

## What this means for you

If you use the core database functionality, you can freely adopt, integrate, and distribute **[ReductStore Core](https://github.com/reductstore/reductstore)** under the Apache 2.0 license. Existing workflows and upgrades remain straightforward.

For teams that rely on advanced functionality or require commercial support, **ReductStore Pro** remains available as the supported commercial edition. We will clearly document which features belong to Core and which are part of Pro in the documentation and release notes.

If you have questions or want to discuss which edition fits your use case, please reach out on the **[ReductStore Community](https://community.reduct.store)** forum or via our **[contact page](/contact)**.
