# Token API Specification Reference

The Token API provides HTTP methods for managing access tokens and permissions for the database.

:::info
The API is available only when the `RS_API_TOKEN` environment variable is set.
:::

## Get a List of Tokens

The method returns a list of tokens with names and creation dates. To use
this method, you need an **[access token](../glossary#access-token)** with full access.

<SwaggerComponent
  method="GET"
  path="/api/v1/tokens"
  summary="Get a list of tokens"
  responses={[
    {
      status: "200",
      message: "OK",
      description: (
        <pre>
          <code>{`{
  "tokens": [
    {
      "name": "string",
      "created_at": "string",
      "last_access": "string | null"
    }
  ]
}`}</code>
        </pre>
      ),
    },
    {
      status: "401",
      message: "Unauthorized",
      summary: "Access token is empty or invalid",
    },
    {
      status: "403",
      message: "Forbidden",
      summary: "Access token doesn't have enough permissions",
    },
    {
      status: "429",
      message: "Too Many Requests",
      summary: "Request rate limit is exceeded",
    },
  ]}
/>

## Show Information about a Token

This method provides full information about a token except its values. The
method requires an access token with full access.

<SwaggerComponent

    "read": [],
    "write": [],
    "ip_allowlist": []

## Create a New Token

The method creates a new access token with given permissions as a JSON
document in the request body. To use this method, you need an access token
with full access.

<SwaggerComponent
  method="POST"
  path="/api/v1/tokens/:token_name"
  summary="Create a new access token"
  parameters={[
    {
      type: "path",
      details: {
        name: "token_name",
        isRequired: true,
        description: "Name of new token",
      },
    },
    {
      type: "body",
      details: {
        name: "full_access",
        dataType: "Boolean",
        isRequired: false,
        description: "Create a token with full access. Default: false",
      },
    },
    {
      type: "body",
      details: {
        name: "read",
        dataType: "String[]",
        isRequired: false,
        description: "A list of bucket names for read access. Default: []",
      },
    },
    {
      type: "body",
      details: {
        name: "write",
        dataType: "String[]",
        isRequired: false,
        description: "A list of bucket names for write access. Default: []",
      },
    },
    {
      type: "body",
      details: {
        name: "ip_allowlist",
        dataType: "String[]",
        isRequired: false,
        description:
          "A list of allowed client IP addresses for this token. Default: [] (no IP restriction)",
      },
    },
  ]}
  responses={[
    {
      status: "200",
      message: "OK",
      summary: "Returns value and timestamp of token",
      description: (
        <pre>
          <code>{`{
  "value": "string",
  "created_at": "string"
}`}</code>
        </pre>
      ),
    },
    {
      status: "401",
      message: "Unauthorized",
      summary: "Access token is empty or invalid",
    },
    {
      status: "403",
      message: "Forbidden",
      summary: "Access token doesn't have enough permissions",
    },
    {
      status: "409",
      message: "Conflict",
      summary: "Token with the same name already exists",
    },
    {
      status: "422",
      message: "Unprocessable Entity",
      summary:
        "Specified bucket does not exist or another validation error occurred",
    },
    {
      status: "429",
      message: "Too Many Requests",
      summary: "Request rate limit is exceeded",
    },
  ]}
/>

### IP allowlist behavior

If `ip_allowlist` is set, the token can be used only from listed client IP addresses.
Leave it empty to allow requests from any IP.

### `last_access` field

Token read endpoints include `last_access`:

- `GET /api/v1/tokens`
- `GET /api/v1/tokens/:token_name`
- `GET /api/v1/me`

The value is populated from audit metadata and can be `null` for tokens that
have not been used yet.

## Rotate a Token

This method rotates a token value (secret) in one explicit API call. The old value
is invalidated immediately after rotation. To use it, a client should have an
access token with full access.

<SwaggerComponent
  method="POST"
  path="/api/v1/tokens/:token_name/rotate"
  summary="Rotate a token"
  parameters={[
    {
      type: "path",
      details: {
        name: "token_name",
        isRequired: true,
        description: "Name of the token to rotate",
      },
    },
  ]}
  responses={[
    {
      status: "200",
      message: "OK",
      summary: "Returns new value and rotation timestamp",
      description: (
        <pre>
          <code>{`{
  "value": "string",
  "created_at": "string"
}`}</code>
        </pre>
      ),
    },
    {
      status: "401",
      message: "Unauthorized",
      summary: "Access token is empty or invalid",
    },
    {
      status: "403",
      message: "Forbidden",
      summary: "Access token doesn't have enough permissions",
    },
    {
      status: "404",
      message: "Not Found",
      summary: "Token with the given name doesn't exist",
    },
    {
      status: "409",
      message: "Conflict",
      summary: "Token cannot be rotated (for example init or provisioned token)",
    },
    {
      status: "429",
      message: "Too Many Requests",
      summary: "Request rate limit is exceeded",
    },
  ]}
/>
