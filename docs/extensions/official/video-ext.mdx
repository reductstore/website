# ReductVideo Extension

The **ReductVideo** [**extension**](../../glossary#extension) exports raw video chunks stored in ReductStore as playable video files.
It accumulates encoded records, detects stream parameters automatically, and muxes them into independently decodable episodes.

## Storage Contract

### H.264

Each record is a segment of H.264 encoded video in **Annex B** byte-stream format (start codes `00 00 01` or `00 00 00 01`).
A record contains one or more NAL units (access units / frames).
Not every record needs to contain a keyframe (IDR) — most will be inter-frames (P/B).
Concatenating all records in timestamp order must produce a valid H.264 elementary stream.

Records must have `content_type: video/h264`.
Timestamps must be the chunk's capture/presentation start time in **microseconds**.

### Labels

| Label | Type     | Mandatory | Description                            |
| ----- | -------- | --------- | -------------------------------------- |
| `fps` | `string` | No        | Frames per second of the stored stream |

Resolution is auto-detected from SPS NAL units in the stream.
If `fps` is not set on records and not overridden via the query parameter, the export will fail with an error.

### Episodes

An **episode** is an output MP4 produced by the exporter. At the start of each episode, the exporter
skips records until it finds one containing an IDR, because an H.264 decoder cannot start without a keyframe.

Episodes can cover the entire query range or be split by `duration` / `size` limits.
Splitting always happens at the next keyframe boundary so that each episode is independently decodable.

### Gap detection

If the timestamp of a record is more than one frame duration beyond the expected next timestamp,
the exporter treats it as a discontinuity. The current episode is finalized and a new one begins,
again waiting for an IDR before accumulating frames.

## Query format

The extension is activated by passing a `video` object inside the `#ext` block of a conditional query:

```python
{
    "#ext": {
        "video": {
            "export": {
                "fps": 30,          # optional: override playback frame rate
                "duration": "1m",   # optional: split episodes by duration
                "size": "100MB"     # optional: split episodes by size
            }
        }
    }
}
```

### Parameters

| Parameter  | Type     | Mandatory | Description                                                                                   |
| ---------- | -------- | --------- | --------------------------------------------------------------------------------------------- |
| `fps`      | `number` | No        | Playback frame rate. Overrides the `fps` label on records.                                    |
| `duration` | `string` | No        | Maximum episode duration (e.g. `"30s"`, `"1m"`, `"1h 30m"`). Split at next keyframe boundary. |
| `size`     | `string` | No        | Maximum episode size (e.g. `"100MB"`, `"1GB"`). Split at next keyframe boundary.              |

:::info
When both `duration` and `size` are specified, the episode is finalized as soon as **either** limit is reached.
:::

## Output

Each output record is a playable MP4 file with `content_type: video/mp4`.
Resolution is extracted from the H.264 SPS NAL unit — no manual configuration is needed.
